Example 1: A Simple File-based Website with Javascript and CSS

This example is a minimal website with an HTML that loads CSS, and Javascript from files.Subsequent examples will add features based on this starting point.

Notable Features

This example can work just by double-clicking or dragging the index.html file into a web browser, or by putting it up on a web server. This is fine for very simple sites that do not require any external resources to be loaded or accessed, as browser security settings prevent it as a safety policy.

  • The entry point is this file, index.html
  • Loads styles from css/style.css that does a css normalize
  • Loads javascript code from js/main.js at the end of the <body>

Workflow Notes

Typically the designing/development cycle is iterative.:

  • Drag the index.html file into a web browser
  • Open any of the files in a text editor such as Visual Studio Code
  • Make changes in the text editor, save changes and then refresh the browser to reload the changed files.
  • See if anything happened, then repeat as many times as necessary.

To debug your work, you would:

  • Open the browser's Javascript Console to see errors and messages from your Javascript code
  • Use the browser's Inspector to see what CSS styles are being successfully applied

Learning the keyboard shortcuts for common operations saves time:

  • Command-S to save
  • Command-Option-J to open the javascript console
  • Command-R to refresh the browser
